## SPECS (Steve's Napkin)

**Big Picture:**
- Designer = section locator. Load a real app, design its sections (S1-S4). Not building the whole app.
- No HEADER component — app already has its own header. Only S1-S4 designable.
- JSON design saves/loads right back where tools were.
- Designer can add injector markers to apps that don't have them — same skill as Factory.

**Layout (2 rows):**
```
Row 1: [folder] | [filename] | "Factory Layout Designer" | [SAVE] [SAVE AS] [☑ JSON Too]
Row 2: [OPEN] | [icons...] [Same Size] [Grp All] | iPad ▼ | Scroll ▼ | Cols/Rows % | CLEAR | Injectors [n] All [n] | >>>
```
- Row 2 scrollable with horizontal slide bar

**Save/Open:**
- SAVE = quick save (no dialog if already saved), first time = acts like SAVE AS
- SAVE AS = always opens explorer
- JSON Too checkbox (default ON) = on save, auto-saves per-section JSON
- OPEN = remembers last folder, loads JSON right back

**JSON Per Section:**
- Saves separate files: -S1.json, -S2.json, -S3.json, -S4.json
- Only sections with tools get saved
- Naming: Factory-MCP-2x2-Designer-Admin-v1.0-S1.json

**Injector Controls (next to CLEAR):**
- Injectors [count] = how many @@@@ markers exist
- All [count] = total sections that should have markers
- All button = auto-add markers for all sections

**Same Size Icon:** Select tools → Same Size → all match first selected
**Group All Icon:** Toggle ON/OFF, ON + drag = everything moves together
**Designer Resource (future):** Tool palette as resource file

## CODING PLAN (One Step at a Time)

### Phase 1: Layout — 5 Rows → 2 Rows
**What:** Restructure the HTML/CSS to collapse 5 toolbar rows into 2.
- Row 1: folder display, filename input, title, SAVE, SAVE AS, JSON Too checkbox
- Row 2: OPEN, tool palette icons, Same Size, Grp All, iPad dropdown, Scroll dropdown, col/row %, CLEAR, injector controls
- Row 2 gets `overflow-x: auto` for horizontal scroll
- Canvas stays as-is below

**Why first:** Everything else builds on the new layout. Get the bones right.

### Phase 2: Save/Open Rework
**What:** Split current SAVE into SAVE + SAVE AS.
- SAVE = if handle exists, write silently. If no handle, act like SAVE AS.
- SAVE AS = always `showSaveFilePicker`
- Both update the filename display in Row 1
- Remove old NAME input — filename comes from save dialog

### Phase 3: Per-Section JSON + JSON Too
**What:** Add checkbox and section-aware JSON save.
- JSON Too checkbox (default ON) in Row 1
- On any SAVE/SAVE AS, if checked, also save per-section JSON
- Reuse existing quad-detection logic (center point of tool → which section)
- Only save sections that have tools
- Naming: base filename + -S1.json, -S2.json, etc.

### Phase 4: Same Size
**What:** New toolbar icon.
- Select 2+ tools → Same Size → all match width/height of first selected
- No distortion of font/perspective (just resize the rect, don't scale text)

### Phase 5: Group All Toggle
**What:** New toolbar icon with ON/OFF state.
- Click = select all tools + group them
- Toggle OFF = ungroup all
- Visual indicator when ON (highlighted icon)
- ON + drag = everything moves together

### Phase 6: Injector Controls
**What:** Replace EXPORT button with injector UI.
- "Injectors [n]" = count of @@@@ markers in current design
- "All [n]" = total sections (always 4)
- All button = auto-add markers to all sections
- Same marker format as Factory uses
