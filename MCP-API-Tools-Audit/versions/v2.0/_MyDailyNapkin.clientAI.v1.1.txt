MODULE 1 - SERVER PRESETS
  Code:      setServerLocal() / setServerProduction()
  Belongs:   api-component2.0.js
  Why:       Every component that talks to the server needs to know
             which server to talk to. This is the URL source of truth.
             api-component owns the connection - not the dashboard,
             not the irrigation component.
---------------------------------------------------------------------
MODULE 2 - TOKEN STAGING
  Code:      setFields(toolName, data)
  Belongs:   api-component2.0.js
  Why:       Staging is the act of preparing a token call - tool name
             + JSON params. This is the api layer's job. It feeds
             directly into the fetch engine (Module 3).
             Dashboard reads results. Irrigation does CRUD.
             Neither one stages tokens - api-component does.
---------------------------------------------------------------------
MODULE 3 - FETCH/SEND ENGINE
  Code:      sendCustomCall()
  Belongs:   api-component2.0.js
  Why:       This IS api-component2.0. The POST fetch to the server,
             the result routing to grid/log - this is the crown jewel.
             Irrigation-component will CALL this engine for its CRUD.
             Dashboard will CALL this engine for display operations.
             Neither replaces it - they consume it.
