crop-client-services — Install on New Machine

First-time setup. Clone the Crop-Client-Services repo from GitHub, install all Node.js dependencies, start APIServer2.0.js on port 3101, and verify it's running. Works on any machine with Git and Node installed.

Reference

Install Steps

1
Git Clone from GitHub
Clone https://github.com/sspappasjr/Crop-Client-Services.git to the target machine. This pulls the entire repo — api-server, pages, tools.
pending
2
NPM Install Dependencies
Run npm install at root level, then again inside api-server/. Installs Express, CORS, and all required packages.
pending
3
Start APIServer2.0.js
Run node APIServer2.0.js from the api-server folder. Should see: 15 tools, port 3101, standalone mode.
pending
4
Verify Server is Running
Ping http://localhost:3101/ping and /tools — confirm it's alive with 15 tools available.

crop-client-services — Rollback to Previous Version

If the deploy went wrong, restore the previous version. Copies the old web.config back and restarts IIS. The old APIServer file is still on the server — just need to point IIS back to it.

Reference

Rollback Steps

1
Restore Previous web.config
RDP to server. Copy web1.6.config → web.config. This points IIS back to APIServer1.6.js.
pending
2
Restart IIS
Run iisreset so IIS picks up the restored web.config and starts the old server file.
pending
3
Verify Rollback Worked
Ping https://api.cropclient.com/ping — confirm server is running the previous version.
pending
