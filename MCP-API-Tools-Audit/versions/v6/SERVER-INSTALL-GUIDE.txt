CROPCLIENT — SERVER INSTALL GUIDE
==================================
Updated: February 16, 2026
Status: Reviewed for accuracy — ready for server deployment

DOMAINS:
  crop-client-services.com     = Website (static pages, landing page, apps)
  api.crop-client-services.com = API Server (port 3101, reverse proxy)

GITHUB REPO:
  https://github.com/sspappasjr/Crop-Client-Services
  Branch: main (alpha)

==============================================
PRE-REQUISITES (on the server)
==============================================
1. Git installed
2. Node.js installed (v18+ recommended)
3. npm comes with Node.js
4. GitHub access (either gh CLI or git credentials)

To verify:
  git --version
  node --version
  npm --version

==============================================
STEP 1: CLONE THE REPO
==============================================
Open CMD or PowerShell on the server.

  git clone https://github.com/sspappasjr/Crop-Client-Services.git
  cd Crop-Client-Services

This is the FRESH INSTALL agent:
  Token: CLONE → from GitHub filing cabinet to server
  Token: STATUS → verify files arrived

Verify:
  dir
  You should see: api-server, Pages, tools, package.json, QUICK_START.txt

==============================================
STEP 2: INSTALL NODE DEPENDENCIES
==============================================
Root level:
  npm install

API server:
  cd api-server
  npm install
  cd ..

==============================================
STEP 3: START THE API SERVER
==============================================
  cd api-server
  node APIServer1.2.js

You should see:
  API MCP Server started
  API MCP Server running on http://localhost:3101
  Tools available: 15

==============================================
STEP 4: VERIFY IT'S RUNNING
==============================================
Open browser or use curl from another terminal:

  A) Ping:        http://localhost:3101/ping
     Expect:      JSON response with status

  B) List Tools:  http://localhost:3101/tools
     Expect:      JSON array of 15 available tools

  C) Test from audit app:
     Open Pages/CropClient-MCP-API-Tools-Audit-Q-v1.4.html in browser
     Set environment to "Local" (http://localhost:3101)
     Click tools grid to test individual endpoints

==============================================
STEP 5: POINT DOMAIN TO SERVER
==============================================
In your domain registrar (GoDaddy):

  For crop-client-services.com (if not already pointed):
  1. A Record: @   → SERVER_IP
  2. A Record: www → SERVER_IP

  For api.crop-client-services.com (API subdomain):
  1. A Record: api → SERVER_IP
  (or CNAME: api → crop-client-services.com)

==============================================
STEP 6: REVERSE PROXY (IIS)
==============================================
Route incoming traffic on api.crop-client-services.com to localhost:3101.

IIS (Windows Server):
  1. Install URL Rewrite module
  2. Install Application Request Routing (ARR) module
  3. Enable ARR proxy in IIS Manager:
     - Server level → Application Request Routing → Server Proxy Settings
     - Check "Enable proxy"
  4. Create new site → api.crop-client-services.com
  5. Add URL Rewrite rule:
     - Inbound rule
     - Pattern: (.*)
     - Rewrite URL: http://localhost:3101/{R:1}

For crop-client-services.com (static site):
  1. Create new site pointing to Crop-Client-Services/Pages folder
  2. Or point to a separate static site folder

==============================================
STEP 7: SSL CERTIFICATE
==============================================
  1. Install Win-ACME or use IIS certificate features
  2. Request cert for crop-client-services.com
  3. Request cert for api.crop-client-services.com
  4. Bind HTTPS to both sites

==============================================
STEP 8: KEEP API SERVER RUNNING
==============================================
API server needs to stay running after you close the terminal.

Option A — PM2 (recommended):
  npm install -g pm2
  cd api-server
  pm2 start APIServer1.2.js --name cropclient-api
  pm2 save
  pm2 startup

Option B — Windows Service:
  Use node-windows or NSSM to register as a service

Option C — Task Scheduler:
  Create scheduled task to run on startup:
  Program: node
  Arguments: C:\path\to\Crop-Client-Services\api-server\APIServer1.2.js

==============================================
STEP 9: FINAL TEST (LIVE DOMAIN)
==============================================
  A) https://api.crop-client-services.com/ping
  B) https://api.crop-client-services.com/tools
  C) Open audit app and set environment to "Online"
     URL: https://api.crop-client-services.com

Both return JSON = you're live.

==============================================
STEP 10: FUTURE UPDATES (GET LATEST)
==============================================
When new code is pushed to GitHub:

  cd C:\path\to\Crop-Client-Services
  git pull origin main

This is the GET LATEST agent:
  Token: PULL → from GitHub filing cabinet to server
  Token: STATUS → confirm clean

Then restart the API server:
  pm2 restart cropclient-api
  (or manually stop/start node)

==============================================
TROUBLESHOOTING
==============================================
Problem: "port 3101 already in use"
  Fix: netstat -ano | findstr 3101
       taskkill /pid <PID> /f

Problem: git clone asks for auth
  Fix: Install gh CLI, run: gh auth login --web
  Or: Use personal access token when prompted

Problem: node not found
  Fix: Install Node.js from https://nodejs.org
  Verify: node --version

Problem: IIS proxy not working
  Fix: Ensure ARR is enabled at server level (not just site level)
  Check: Application Request Routing → Server Proxy Settings → Enable proxy = checked
