  - DELETE the placeholders

It just never got extracted properly. Today that gets fixed.

WHAT GETS CREATED — irrigation-component.js
=====================================================================

New standalone file:
  v2.0-apiupdate/TestHarness/AIToolKit/irrigation-component.js

Source of truth for content:
  v1.6-restructure/CropClient-MCP-API-Tools-Audit-Q-v1.6.html
  Extract verbatim from between the @@@@ IRRIGATION_COMPONENT markers.

Structure:
  // ============================================================
  // IRRIGATION_COMPONENT v1.1
  // ============================================================
  // Pure MCP tokens for irrigation domain operations
  // NO DOM — params in, { returnCode, statusMessage, data } out
  // Injectable via @@@ markers into any harness or component
  // Source: CropClient-MCP-API-Tools-Audit-Q-v1.6.html
  // ============================================================

  // @@@ BEGIN irrigation-component @@@

  [date helpers]
    parseEventDate(dateStr)
    formatDate(date)

  [4 CRUD tokens — original names, original logic]
    testCreateNext(ranch, planting, records)
    resetTable(sourceData, targetRecords, dataSourceFlag)
    testReadMeter(ranch, planting, records)
    updateRecord(recordId, patch, records)

  [2 JSON persistence tokens — were missing, now restored]
    saveDisplay(records)
    retrieveDisplay(jsonData, targetRecords)

  // @@@ END irrigation-component @@@

Contract (non-negotiable):
  - NO DOM access anywhere in this file
  - Every function: params in → { returnCode, statusMessage, data } out
  - returnCode 0 = success, negative = error
  - statusMessage is human-readable for the log
  - data is the record or result object

This file is self-contained and injectable into any harness
via a script tag or copy-paste between @@@ markers.
