RULE: web.config VERSIONED PAIR
=============================================
  IIS requires the file be named "web.config" — cannot rename it.
  web2.0.config is an EXACT COPY so we always know what version is live.
  They MUST stay in sync. When you change web.config, update web2.0.config.
  When you deploy a new version, create web(version).config as the pair.

  Example:
    web.config      ← what IIS reads (always this name)
    web2.0.config   ← exact copy (tells us production = v2.0)
